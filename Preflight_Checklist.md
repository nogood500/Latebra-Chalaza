# Latebra Chalaza 公開前チェックリスト

**【目的】**
このサイトは静的HTMLであり、バックエンド（データベース等）からの情報漏洩リスクはゼロです。
唯一かつ最大のリスクは、**「手動コピペ運用時に、意図しない個人情報をHTMLに貼り付けて公開してしまうこと（ヒューマンエラー）」**です。
これを防ぐため、GitHubへPushする前に必ず以下の3点を確認します。

---

## 🛑 公開前の絶対確認事項（3項目）

- [ ] **1. 個人情報・固有名詞の排除**
    *   生徒の名前、保護者の名前、特定のクラス名、職場（智学館）に関する非公開情報が含まれていないか？
    *   日付や場所の組み合わせで、個人が特定できる行動履歴になっていないか？

- [ ] **2. 家族のプライバシー保護**
    *   妻の具体的な病状や、子供（藍加・息子）の学校トラブルに関する過度にセンシティブな生々しい記述が含まれていないか？（抽象化された「現象」としての記述は可）

- [ ] **3. ソースコード内の不用意なコメントの確認**
    *   `index.html` や `style.css` の中に、公開を意図しない私的なメモ（HTMLコメント `<!-- -->` 等）が残っていないか？

---

## 🚀 デプロイ / 更新手順

1. Obsidian上で `#public` タグが付与されたZettel（公開してよいと判断したもの）を選ぶ。
2. `index.html` の `<main class="stream">` 内に、専用のフォーマットで貼り付ける。
    *   *※ `--drift` 変数を手動で微調整して「よろめき」を持たせること。*
3. 上記の**「公開前チェックと確認」**を完了する。
4. Gitコミットし、GitHubの `main` ブランチへPushする。
    *   `git add .`
    *   `git commit -m "Add new Zettel (YYYY-MM-DD)"`
    *   `git push origin main`
    *   *(※ 数分以内に GitHub Pages 側で自動的に公開・反映されます)*
